# 批量下载功能修复说明

## 问题诊断

批量下载功能可能无法工作的原因：

1. **文章列表获取失败** - 无法从公众号主页提取文章列表
2. **页面加载超时** - 批量打开标签页时页面加载太慢
3. **内容提取失败** - content script 注入或提取失败
4. **权限问题** - 缺少必要的扩展权限

## 修复方案

### 1. 改进文章列表获取

已优化 `fetchArticleList()` 函数，使用多种方法提取文章：
- 从 DOM 中提取链接
- 从页面源码中提取
- 从 JavaScript 变量中提取
- 自动滚动加载更多内容

### 2. 改进批量下载流程

- 添加重试机制（最多3次）
- 改进页面加载等待逻辑
- 添加延迟避免请求过快
- 更好的错误处理和提示

### 3. 使用说明

#### 方法一：从文章页面批量下载

1. 打开任意一篇公众号文章
2. 点击扩展图标
3. 切换到"批量下载"模式
4. 点击"获取该作者的所有文章"
5. 等待文章列表加载
6. 选择要下载的文章（默认全选）
7. 点击"批量下载选中文章"

#### 方法二：从公众号主页批量下载

1. 访问公众号主页（需要登录微信）
2. 手动滚动页面加载更多文章
3. 点击扩展图标
4. 切换到"批量下载"模式
5. 点击"获取该作者的所有文章"
6. 选择并下载

## 调试步骤

如果批量下载仍然无法工作，请按以下步骤调试：

### 1. 检查控制台错误

**Popup 控制台：**
- 右键扩展图标 → 检查弹出内容
- 查看 Console 标签中的错误

**Content Script 控制台：**
- 在文章页面按 F12
- 查看 Console 标签中的错误

### 2. 检查文章列表获取

1. 点击"获取该作者的所有文章"
2. 查看状态提示
3. 如果显示"未找到文章列表"，尝试：
   - 刷新页面
   - 手动滚动公众号主页
   - 确保已登录微信

### 3. 检查批量下载过程

1. 开始批量下载后，查看进度条
2. 如果卡在某个文章，查看控制台错误
3. 检查是否有网络问题

## 常见问题

### Q: 点击"获取文章列表"后没有反应

**A:** 可能的原因：
1. 不在正确的页面（需要公众号文章或主页）
2. 页面未完全加载
3. 公众号主页需要登录

**解决方法：**
- 刷新页面后重试
- 确保在微信公众号页面
- 尝试手动滚动页面后再获取

### Q: 获取到文章列表但下载失败

**A:** 可能的原因：
1. 文章链接无效
2. 页面加载超时
3. 内容提取失败

**解决方法：**
- 检查网络连接
- 减少批量下载数量
- 查看控制台错误信息

### Q: 批量下载很慢

**A:** 这是正常的，因为：
- 需要逐个打开文章页面
- 需要等待页面加载
- 需要提取内容

**优化建议：**
- 减少批量下载数量（每次10-20篇）
- 使用较快的网络连接
- 关闭其他占用资源的标签页

## 技术细节

### 批量下载流程

1. **获取文章列表** - 从当前页面提取所有文章链接
2. **选择文章** - 用户选择要下载的文章
3. **逐个下载** - 对每篇文章：
   - 创建新标签页（后台）
   - 等待页面加载完成
   - 注入 content script
   - 提取文章内容
   - 生成文件并下载
   - 关闭标签页
   - 延迟避免请求过快

### 错误处理

- 每篇文章最多重试3次
- 失败的文章会记录但继续下载其他文章
- 下载完成后显示成功/失败统计

### 性能优化

- 使用后台标签页（active: false）
- 添加延迟避免请求过快（1.5-2秒）
- 及时关闭已完成的标签页
- 限制最大下载数量（100篇）
